luks-agent decision flow

Agent scans volumes, checks encryption flag, routes to Path A (new volume) or Path B (existing encrypted volume)

flowchart TD T1["Trigger 1: boot.service\n(every boot)"] --> SCAN["Scan all /dev/vd(b..z)\nGet Cinder UUID via lsblk SERIAL"] T2["Trigger 2: udev rule\n(hot-attach)"] --> SINGLE["Single new device\nGet Cinder UUID via lsblk SERIAL"] SCAN --> QUERY["Query Custom Metadata Server\nGET /v1/volumes/UUID"] SINGLE --> QUERY QUERY --> CHECK_FLAG{"encrypt = true?"} CHECK_FLAG -->|No| SKIP["Skip this volume"] CHECK_FLAG -->|Yes| CHECK_LUKS{"cryptsetup isLuks?"} CHECK_LUKS -->|"No — new volume"| PATH_A CHECK_LUKS -->|"Yes — existing volume"| PATH_B subgraph PATH_A["Path A: First-time encryption"] A1["Generate random passphrase\nhead -c 32 /dev/urandom\n(memory only)"] A1 --> A2["POST OpenBao transit/encrypt/luks-master\nGet ciphertext back"] A2 --> A3["PUT ciphertext to OpenBao KV\nluks/data/volumes/UUID"] A3 --> A4["cryptsetup luksFormat --type luks2"] A4 --> A5["cryptsetup open\nmkfs.ext4\nmount /mnt/UUID"] A5 --> A6["Backup LUKS header to OpenBao KV"] A6 --> A7["Wipe passphrase from memory\nUpdate /etc/fstab"] end subgraph PATH_B["Path B: Unlock existing"] B1["GET ciphertext from OpenBao KV\nluks/data/volumes/UUID"] B1 --> B2["POST OpenBao transit/decrypt/luks-master\nGet passphrase back"] B2 --> B3["cryptsetup open\nmount /mnt/UUID"] B3 --> B4["Wipe passphrase from memory"] end
Detailed sequence diagrams

Path A — First-time encryption

New volume: generate passphrase, encrypt via Transit, store in KV, create LUKS header, mount

sequenceDiagram participant Agent as luks-agent participant TR as OpenBao Transit participant KV as OpenBao KV participant VOL as Volume /dev/vdX Note over Agent: encrypt=true, isLuks=No → new volume Agent->>Agent: Generate passphrase
head -c 32 /dev/urandom (memory only) Agent->>TR: POST transit/encrypt/luks-master
{plaintext: base64(passphrase)} Note over TR: Encrypts with internal key
Stores nothing TR-->>Agent: ciphertext vault:v1:8SD... Agent->>KV: PUT luks/data/volumes/VOL_UUID
{ciphertext: vault:v1:8SD...} Note over KV: Stored permanently by volume UUID Agent->>VOL: cryptsetup luksFormat --type luks2 --key-file - Note over VOL: LUKS header created Agent->>VOL: cryptsetup open Agent->>VOL: mkfs.ext4 + mount /mnt/UUID Agent->>KV: Backup LUKS header Note over Agent: unset PASS — passphrase exists nowhere

Path B — Unlock existing volume

Existing encrypted volume: retrieve ciphertext from KV, decrypt via Transit, unlock and mount

sequenceDiagram participant Agent as luks-agent participant KV as OpenBao KV participant TR as OpenBao Transit participant VOL as Volume /dev/vdX Note over Agent: encrypt=true, isLuks=Yes → existing volume Agent->>KV: GET luks/data/volumes/VOL_UUID KV-->>Agent: ciphertext vault:v1:8SD... Agent->>TR: POST transit/decrypt/luks-master
{ciphertext: vault:v1:8SD...} TR-->>Agent: plaintext passphrase Agent->>VOL: cryptsetup open --key-file - Agent->>VOL: mount /mnt/UUID Note over Agent: unset PASS — volume stays mounted

Token renewal

Systemd timer fires every 45 minutes; VM calls IMDS /v1/renew; IMDS signs fresh JWT and exchanges for new token

sequenceDiagram participant Timer as systemd timer participant VM as Guest VM participant IMDS as IMDS participant BAO as OpenBao loop Every 45 minutes Timer->>VM: Fire luks-token-renew.service VM->>IMDS: POST /v1/renew Note over IMDS: Verify VM by source IP
Look up instance-id from IP
Sign new JWT (RSA-4096) IMDS->>BAO: POST /auth/jwt/login with new JWT BAO-->>IMDS: Fresh token (1h TTL) IMDS-->>VM: New token VM->>VM: Write to /run/bao-token (0600) Note over VM: Token refreshed
Next LUKS operation uses new token end
